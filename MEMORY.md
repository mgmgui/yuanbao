# MEMORY.md - 长期记忆与更新策略

## 1. 长期记忆定位

本文件只保存可长期复用的事实、规则和关键结论。
日常执行细节写入 memory/YYYY-MM-DD.md。

## 2. 当前长期事实

- 用户目标: 建立高质量、可落地、可审计的小说项目交付体系。
- 核心要求: 先提案后开发；所有流程必须有严格 SOP 与驳回机制。
- 平台目标: 双平台交付（微信公众号 + 小红书）并行输出。
- 执行原则: 先本地自修，再 issue 升级。

## 3. 历史业务基线

- 已存在较多人物档案、小说正文、评审报告、监控脚本与定时任务配置。
- 小说项目已建立监控节奏（5/10/30/60 分钟不同维度巡检）。
- 部分作品已有评分记录，存在需重写项。

## 4. 记忆写入触发器（强制）

出现以下情况，必须写入 memory/YYYY-MM-DD.md:

- 关键决策（流程、目录、门禁、升级策略）
- 严重问题与升级事件
- 被验证有效的修复策略
- 被驳回并重写的典型原因
- 启动回归异常与自修处理结果（含是否升级 issue）

出现以下情况，必须同步更新本文件:

- 可长期复用的规则稳定下来
- 用户偏好发生明确变化
- 项目治理模型发生结构性调整

## 5. 维护节奏

- 每次主会话结束前: 更新当日 memory 文件
- 每周至少一次: 从日记忆提炼更新 MEMORY.md
- 每次 startup 自检失败后: 在当日 memory 追加一条“回归-处理-结果”记录
