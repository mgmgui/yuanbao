# 仓库治理规则

## 1. 目标

保证仓库可导航、可协作、可审计，避免根目录持续堆积与文件失控。

## 2. 落位原则

- 新文件必须按类型进入 canonical 目录。
- 根目录仅保留运行入口、核心配置、身份与导航文件。
- 历史产出必须归档，禁止长期停留在根目录。

## 3. 反混乱约束

- 禁止新增“未分类”目录。
- 禁止在根目录新增一次性报告。
- 禁止无版本号覆盖关键评审与交付文档。

## 4. 变更检查

每次提交前至少检查:

- 文件是否在正确目录
- 文件名是否符合命名规范
- 是否需要同步更新 archive 索引
- 是否需要记录迁移映射

## 5. 审查责任

- 提交者: 对文件落位和命名负责
- 评审者: 对治理约束合规性负责
- 运营者: 维护归档索引与迁移日志
