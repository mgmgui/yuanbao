# 命名规范（冻结版）

生效日期: 2026-03-28

## 1. 基本规则

- 统一使用小写英文与连字符命名目录与英文文件。
- 中文业务文档允许保留中文文件名，但必须包含清晰后缀语义。
- 禁止使用“最终版2/新新版/临时”这类不可追踪命名。

## 2. 推荐命名模式

- 项目文档:
  - <topic>-<stage>-v<major>.<minor>.md
- 报告文档:
  - <topic>-report-<yyyy-mm-dd>.md
- 模板文档:
  - <topic>-template.md
- 迁移日志:
  - migration-batch-<n>-<yyyy-mm-dd>.md

# 仓库治理规则

## 1. 目标

保证仓库可导航、可协作、可审计，避免根目录持续堆积与文件失控。

## 2. 落位原则

- 新文件必须按类型进入 canonical 目录。
- 根目录仅保留运行入口、核心配置、身份与导航文件。
- 历史产出必须归档，禁止长期停留在根目录。

## 3. 反混乱约束

- 禁止新增“未分类”目录。
- 禁止在根目录新增一次性报告。
- 禁止无版本号覆盖关键评审与交付文档。

## 4. 变更检查

每次提交前至少检查:

- 文件是否在正确目录
- 文件名是否符合命名规范
- 是否需要同步更新 archive 索引
- 是否需要记录迁移映射

## 5. 审查责任

- 提交者: 对文件落位和命名负责
- 评审者: 对治理约束合规性负责
- 运营者: 维护归档索引与迁移日志
