# 每日/每周执行节拍表（SOP 2.0）

## 1. 目标

通过固定节拍把交付流程从“人盯人”变成“节拍盯流程”，确保高质量交付可复制、可审计、可预警。

执行原则:
- 同步节拍: 每日按固定时间推进工序，不随个人习惯波动。
- 门禁节拍: 每个关键窗口都要做门禁检查，不通过即停线。
- 闭环节拍: 每个拒绝、阻塞、升级都必须在下一节拍窗口被处理。

## 4. 事件驱动节拍（非固定时段）

- 驳回事件:
  - T+0: 生成驳回包并通知 R/A/PM/PO。
  - T+4h: 重写人提交修订计划。
  - T+24h: 完成重写并回归评审（超时自动升级）。
- 阻塞事件:
  - 超过 24h 未解除 -> 进入阻塞升级。
  - 连续 2 次 heartbeat 检测到同一阻塞 -> 强制 issue 化。
- 质量异常事件:
  - 同类问题连续 3 次驳回 -> 触发专项工艺复盘会。

## 5. 关键KPI与节拍阈值

- 工序准时率 >= 95%
- 一审通过率 >= 70%（低于阈值触发写作训练）
- 关键项一次通过率 >= 90%
- 驳回闭环时效 <= 24h
- 路径命名合规率 = 100%
- 双平台发布包合格率 = 100%
