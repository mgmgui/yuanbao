# 全球人物小说项目行动规则

## 基本行动原则

1. **全权代理推进**：用户授权我全权代替推进全球人物小说项目
2. **四Agent流水线**：严格按照四Agent流程执行每一篇小说创作
3. **一篇一篇交付**：按TODO清单一篇一篇推进，交付一篇就标记完成一篇
4. **质量优先**：必须达到矛盾标准和拿奖标准才能交付

## 四Agent流水线详细流程

### Agent 1：构思人物Agent
**任务**：完成人物档案
**输出**：人物档案文件
**标准**：
- 人物基本信息完整
- 历史角色定位清晰
- 矛盾特性分析到位
- 适合小说创作的人物特点

### Agent 2：构思背景Agent
**任务**：补充历史背景资料
**输出**：背景分析文档
**标准**：
- 历史时代背景详实
- 社会环境分析到位
- 人物所处历史环境准确
- 适合小说创作的历史舞台

### Agent 3：构思创意Agent
**任务**：补充小说创意点子
**输出**：创意构思文档
**标准**：
- 小说主题清晰
- 情节框架合理
- 人物塑造计划完整
- 冲突设计丰富
- 文学性构思深入

### Agent 4：构思小说Agent
**任务**：创作小说正文
**输出**：小说正文文件
**标准**：
- 完整的小说章节
- 符合人物档案设定
- 体现矛盾性和复杂性
- 文学性强，情节引人入胜
- 达到拿奖标准

## 评审标准

### 矛盾标准（必须达标）
1. 人物内心矛盾展现充分
2. 历史角色冲突设计合理
3. 性格复杂性刻画深刻
4. 人物多面性表现到位

### 拿奖标准（必须达标）
1. 情节完整且引人入胜
2. 人物塑造立体且深刻
3. 文学性强（语言优美、结构严谨）
4. 思想深刻（有深度主题）
5. 结构严谨（章节安排合理）

## 驳回和修改流程

### 评审不达标情况：
1. 人物矛盾性不足 → 退回构思人物Agent
2. 历史背景不足 → 退回构思背景Agent
3. 创意构思不足 → 退回构思创意Agent
4. 小说质量不足 → 退回构思小说Agent

### 修改要求：
- 必须明确指出不足处
- 提供修改方向和具体建议
- 重新执行对应Agent流程
- 再次评审直到达标

## 交付流程

### 交付条件：
1. 四Agent流程全部完成
2. 评审达标（符合矛盾标准和拿奖标准）
3. 所有相关文件齐全
4. 归档文件完整

### 交付动作：
1. 交付小说文件给用户
2. 在TODO清单中标记完成
3. 在项目进度表中更新状态
4. 记录交付时间和评审结果

## 项目管理规则

### TODO清单推进原则：
1. 按清单顺序一篇一篇推进
2. 完成一篇才开始下一篇
3. 每篇必须完整执行四Agent流程
4. 交付后才能标记完成

### 进度报告：
1. 每完成一个Agent阶段报告
2. 评审结果报告
3. 交付完成报告
4. 整体进度报告

## 文件管理规则

### 文件命名规范：
1. 人物档案：人物名_人物档案.md
2. 背景构思：人物名_背景构思.md
3. 创意构思：人物名_创意构思.md
4. 小说正文：人物名_小说.md
5. 评审报告：人物名_评审报告.md
6. 归档文件：人物名_归档.md
